# Инструкция по тестированию исправления изображений

## Подготовка к тестированию

### 1. Дождаться завершения деплоя
Проверить статус деплоя на Render Dashboard:
https://dashboard.render.com/web/srv-d8csddt7vvec73ckovt0

Деплой завершен, когда:
- Статус: "Live"
- Зеленая галочка на последнем коммите
- В логах: "🚀 Server running on port 10000"

### 2. Проверить, что новый код задеплоен
```bash
# Проверить, что /uploads/ доступен (должен вернуть 403 или список файлов)
curl -I https://student-connect-backend.onrender.com/uploads/

# Если вернул 404 - старая версия еще работает
# Если вернул 403 Forbidden - новая версия задеплоена (папка пустая, но маршрут есть)
```

## Тестирование на двух устройствах

### Сценарий 1: Пост с одним изображением

**Устройство 1:**
1. Запустить приложение: `flutter run -d <device-1-id>`
2. Войти в аккаунт (или зарегистрироваться)
3. Нажать "+" → "Новый пост"
4. Добавить текст: "Тест изображения"
5. Добавить 1 фото из галереи
6. Нажать "Опубликовать"
7. ✅ Проверить, что фото отображается в ленте

**Устройство 2:**
1. Запустить приложение: `flutter run -d <device-2-id>`
2. Войти в другой аккаунт
3. Обновить ленту (потянуть вниз)
4. ✅ Проверить, что пост с фото от Устройства 1 отображается
5. ✅ Проверить, что фото загружается и видно

### Сценарий 2: Пост с несколькими изображениями

**Устройство 1:**
1. Создать пост с 3-5 фотографиями
2. ✅ Проверить, что все фото отображаются в карусели
3. ✅ Проверить, что индикатор страниц работает

**Устройство 2:**
1. Обновить ленту
2. ✅ Проверить, что все фото загружаются
3. ✅ Проверить, что можно листать фото

### Сценарий 3: Проект с изображениями

**Устройство 1:**
1. Нажать "+" → "Новый проект"
2. Заполнить название и описание
3. Добавить 2-3 фото
4. Добавить навыки
5. Опубликовать проект
6. ✅ Проверить, что фото отображаются

**Устройство 2:**
1. Перейти на вкладку "Поиск"
2. Найти проект
3. ✅ Проверить, что все изображения загружаются

### Сценарий 4: Просмотр изображения в полном размере

**Любое устройство:**
1. Открыть пост с изображением
2. Нажать на изображение
3. ✅ Проверить, что открывается полноэкранный просмотр
4. ✅ Проверить, что можно листать (если несколько фото)
5. ✅ Проверить, что можно закрыть

## Проверка логов

### Backend логи на Render
1. Открыть: https://dashboard.render.com/web/srv-d8csddt7vvec73ckovt0
2. Перейти на вкладку "Logs"
3. Искать строки:
   ```
   POST /api/posts
   Post created with images: ["/uploads/post_..."]
   ```

### Flutter логи
В консоли при создании поста должно быть:
```
D/ApiService: POST /api/posts
D/ApiService: Response: {"post": {..., "images": ["https://..."]}}
```

## Что делать, если не работает

### Изображение не загружается на Устройстве 2
1. Проверить URL изображения в логах Flutter
2. Должен быть: `https://student-connect-backend.onrender.com/uploads/...`
3. Если локальный путь (`/data/user/...`) - старая версия кода

### Ошибка при создании поста
1. Проверить логи backend на Render
2. Проверить размер изображения (должен быть < 10MB)
3. Проверить формат (jpg, png, gif, webp)

### 404 на /uploads/
1. Деплой еще не завершен - подождать
2. Или откатиться на старый коммит
