# 🎯 Отчет: Исправление отображения изображений постов

## Дата: 2026-06-02

---

## 📋 Задача

**Проблема:** Фото поста, выложенного с одного устройства, не отображается на другом устройстве. Сам пост виден, но изображения нет.

**Корневая причина:** Flutter передавал локальные файловые пути (например, `/data/user/0/.../post_123.jpg`) на backend, который просто сохранял их в базе данных. Когда другое устройство получало пост, оно пыталось загрузить изображение по несуществующему локальному пути.

## 🔧 Технические детали

### До исправления:
```
Устройство 1 → Локальный путь (/data/...) → Backend → База данных
                                                              ↓
Устройство 2 ← Локальный путь (/data/...) ← Backend ← База данных
❌ Файл не существует на Устройстве 2
```

### После исправления:
```
Устройство 1 → Файл (FormData) → Backend → Сохранить в uploads/ → База данных (/uploads/...)
                                                                            ↓
Устройство 2 ← Полный URL (https://.../uploads/...) ← Backend ← База данных
✅ Файл загружается с сервера
```

### Формат хранения в БД:
```json
{
  "images": ["/uploads/post_1717349925-uuid.jpg"]
}
```

### Формат в Flutter:
```dart
images: ["https://student-connect-backend.onrender.com/uploads/post_1717349925-uuid.jpg"]
```

## ⚠️ Важные замечания

### Ограничения Render Free Plan:
1. **Папка uploads/ не персистентная** - очищается при каждом деплое
2. Для production рекомендуется:
   - Платный план Render с persistent storage
   - Или облачное хранилище (AWS S3, Cloudinary, Firebase Storage)

### Старые посты:
- Посты, созданные до этого исправления, будут иметь локальные пути
- Они не будут отображаться
- Решение: удалить вручную или создать миграцию (не требуется для диплома)

### Размер и лимиты:
- Максимум 5 изображений в одном посте
- Максимум 10MB на изображение
- Форматы: JPG, PNG, GIF, WEBP
