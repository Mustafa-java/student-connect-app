1. ВВЕДЕНИЕ И КОНЦЕПЦИЯ ПРОЕКТА
═══════════════════════════════════════════════════════════════════════════════

В рамках дипломной работы я разработал мобильное приложение "СтудХаб" —
социальную платформу для студентов, объединяющую функционал социальной сети,
портфолио проектов и мессенджера. Приложение создано на Flutter и имеет
современный дизайн, вдохновленный Instagram 2025.

Основная идея проекта — создать единое пространство, где студенты могут:
• Демонстрировать свои проекты и достижения
• Находить единомышленников для совместной работы
• Общаться и обмениваться опытом
• Следить за работами других студентов
• Скачивать исходный код проектов для обучения

3. BACKEND И API
═══════════════════════════════════════════════════════════════════════════════

Я разработал полноценный REST API на Node.js + Express для работы приложения:

✓ Технологический стек backend:
  - Node.js + Express.js — веб-сервер
  - SQLite (sql.js) — база данных
  - JWT — аутентификация
  - bcrypt — хеширование паролей
  - multer — загрузка файлов
  - CORS — кросс-доменные запросы

✓ Реализованные API эндпоинты:

  Аутентификация:
  • POST /api/auth/register — регистрация нового пользователя
  • POST /api/auth/login — вход в систему
  • GET /api/auth/me — получение текущего пользователя

  Пользователи:
  • GET /api/users — список всех пользователей
  • GET /api/users/:id — информация о пользователе
  • PUT /api/users/:id — обновление профиля
  • GET /api/users/search?q=query — поиск пользователей

  Посты:
  • GET /api/posts — получение всех постов
  • POST /api/posts — создание нового поста
  • DELETE /api/posts/:id — удаление поста
  • POST /api/posts/:id/like — лайк/анлайк поста
  • GET /api/posts/:id/comments — комментарии к посту
  • POST /api/posts/:id/comments — добавление комментария

  Проекты:
  • GET /api/projects — получение всех проектов
  • POST /api/projects — создание нового проекта
  • DELETE /api/projects/:id — удаление проекта
  • POST /api/projects/:id/like — лайк/анлайк проекта
  • POST /api/projects/:id/views — увеличение счетчика просмотров
  • POST /api/projects/:id/upload-zip — загрузка ZIP-файла
  • GET /api/projects/:id/zip-file — скачивание ZIP-файла

  Чаты и сообщения:
  • GET /api/chats — список чатов пользователя
  • POST /api/chats — создание нового чата
  • GET /api/chats/:id/messages — сообщения в чате
  • POST /api/chats/:id/messages — отправка сообщения
  • POST /api/chats/:id/read — отметить чат как прочитанный

  Подписки:
  • POST /api/follow/:userId — подписаться/отписаться
  • GET /api/follow/status/:userId — статус подписки
  • GET /api/followers/:userId — список подписчиков
  • GET /api/following/:userId — список подписок

✓ База данных SQLite:
  - Таблица users — пользователи
  - Таблица posts — посты
  - Таблица projects — проекты
  - Таблица comments — комментарии
  - Таблица likes — лайки
  - Таблица chats — чаты
  - Таблица messages — сообщения
  - Таблица follows — подписки
  - Таблица project_files — файлы проектов
