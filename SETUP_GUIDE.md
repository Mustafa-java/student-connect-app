## 🔧 Установка Flutter

### Шаг 1: Скачивание Flutter SDK

1. Перейди на официальный сайт Flutter: https://docs.flutter.dev/get-started/install/windows
2. Нажми **"Download the installer"** (кнопка для Windows)
3. Запусти скачанный файл `flutter_windows_X.X.X-stable.zip`

### Шаг 2: Распаковка Flutter

1. Распакуй архив в удобное место, например: `C:\src\flutter`
2. **Важно:** Не распаковывай в папки, требующие прав администратора (например, `C:\Program Files`)

### Шаг 3: Добавление Flutter в PATH

1. Нажми `Win + R`, введи `sysdm.cpl`, нажми Enter
2. Перейди во вкладку **"Дополнительно"** → **"Переменные среды"**
3. В разделе **"Переменные среды пользователя"** найди переменную `Path`
4. Нажми **"Изменить"** → **"Создать"**
5. Добавь путь: `C:\src\flutter\bin` (или тот путь, куда ты распаковал Flutter)
6. Нажми **OK** и закрой все окна

### Шаг 4: Проверка установки

1. Открой **новую** командную строку (Win + R → `cmd` → Enter)
2. Введи команду:
   ```bash
   flutter doctor
   ```
3. Ты увидишь список компонентов. Не пугайся крестиков — сейчас всё исправим!

### Шаг 5: Установка недостающих компонентов

`flutter doctor` покажет, что нужно установить. Обычно это:

- ✅ **Flutter SDK** — уже установлен
- ❌ **Android toolchain** — нужно установить Android Studio или cmdline-tools
- ❌ **Chrome** — опционально, для веб-разработки
- ❌ **Android Studio** — нужно для эмулятора

## 📱 Установка эмулятора Android

### Вариант 1: Android Studio (стандартный)

#### Шаг 1: Установка Android Studio

1. Перейди на сайт: https://developer.android.com/studio
2. Нажми **"Download Android Studio"**
4. Запусти установщик и следуй инструкциям
5. При установке выбери компоненты:
   - ✅ Android Studio
   - ✅ Android Virtual Device (AVD)
   - ✅ Android SDK
   - ✅ Android SDK Platform

#### Шаг 2: Настройка Android SDK

1. Открой Android Studio
2. Перейди в **Tools** → **SDK Manager**
4. Отметь галочками:
   - Android SDK Platform (последнюю версию, например Android 14 API 34)
   - Android SDK Build-Tools
   - Android SDK Command-line Tools
5. Нажми **Apply** → **OK**

#### Шаг 3: Создание виртуального устройства

1. В Android Studio перейди в **Tools** → **Device Manager**
2. Нажми **"Create Device"**
3. Выбери устройство (например, **Pixel 5**)
4. Нажми **Next**
5. Выбери системный образ:
   - Рекомендую: **API 34 (Android 14)** с **x86 Images**
   - Если нет — нажми **"Download"** рядом с образом
6. Нажми **Next** → **Finish**

#### Шаг 4: Запуск эмулятора

1. В Device Manager нажми ▶️ напротив созданного устройства
2. Подожди загрузки (первый запуск может занять 2-5 минут)
3. Готово! Эмулятор работает!

### Вариант 2: Genymotion (быстрая альтернатива)

#### Шаг 1: Установка Genymotion

1. Перейди на сайт: https://www.genymotion.com/download/
3. Скачай версию **Genymotion Desktop** для своей ОС
4. Запусти установщик и установи программу

#### Шаг 2: Регистрация и вход

1. Открой Genymotion
2. Войди в свой аккаунт (или создай новый)
3. Бесплатный тариф: **Personal Use** (для личного использования)

#### Шаг 3: Создание виртуального устройства

1. Нажми **"Add"** (плюс)
2. Выбери версию Android (рекомендую **Android 13** или **14**)
3. Выбери шаблон устройства (например, **Pixel 4**)
4. Нажми **Next** → **Install**
5. Подожди загрузки образа

#### Шаг 4: Запуск эмулятора

1. Нажми ▶️ напротив созданного устройства
2. Genymotion запускается быстрее стандартного эмулятора!

#### Шаг 5: Интеграция с Flutter

1. Убедись, что эмулятор запущен
2. В командной строке введи:
   ```bash
   flutter devices
   ```
3. Ты должен увидеть своё устройство Genymotion в списке

### Вариант 3: Запуск на физическом телефоне

#### Шаг 1: Включение режима разработчика

**На телефоне Android:**

1. Открой **Настройки**
2. Перейди в **"О телефоне"** (или **"Сведения о телефоне"**)
3. Найди **"Номер сборки"** (или **"Build number"**)
4. Нажми на него **7 раз подряд**
5. Появится сообщение: **"Вы стали разработчиком!"** 🎉

#### Шаг 2: Включение отладки по USB

1. Вернись в главное меню настроек
2. Перейди в **"Система"** → **"Для разработчиков"**
3. Включи опцию **"Отладка по USB"** (USB Debugging)
4. Подтверди действие

#### Шаг 3: Подключение к компьютеру

1. Подключи телефон к компьютеру через USB-кабель
2. На телефоне появится запрос на разрешение отладки — нажми **"Разрешить"**
4. Введи команду:
   ```bash
   flutter devices
   ```
5. Ты должен увидеть свой телефон в списке!

#### Шаг 4: Запуск на телефоне

```bash
flutter run
```

Приложение установится и запустится на твоём телефоне!

## ▶️ Запуск приложения

### Способ 1: Через командную строку

```bash
flutter run
```

Приложение скомпилируется и запустится на активном устройстве.

### Способ 2: На конкретном устройстве

1. Узнай ID устройства:
   ```bash
   flutter devices
   ```
2. Запусти на нужном устройстве:
   ```bash
   flutter run -d <device-id>
   ```
   Например: `flutter run -d emulator-5554`

### Способ 3: В режиме отладки с консолью

```bash
flutter run -v
```

Флаг `-v` покажет подробный лог (полезно при ошибках).

### Способ 4: Через VS Code

1. Открой папку проекта в VS Code
2. Установи расширение **Flutter** (если не установлено)
3. В нижней панели выбери устройство
4. Нажми **F5** или кнопку **▶️ Run**

### Способ 5: Сборка APK для установки на телефон

Чтобы создать APK-файл и отправить другу:

```bash
flutter build apk --release
```

APK-файл будет находиться в:
```
build\app\outputs\flutter-apk\app-release.apk
```

Этот файл можно установить на любой Android-телефон!

## 🛠 Частые ошибки и решения

### Ошибка 1: "No devices found"

**Причина:** Нет запущенного эмулятора или подключённого телефона.

**Решение:**
```bash
flutter emulators                    # Показать список эмуляторов
flutter emulators --launch <id>      # Запустить эмулятор
flutter devices                      # Проверить устройства
```

### Ошибка 2: "Build failed" или "Gradle error"

**Причина:** Проблемы с зависимостями или кэшем.

**Решение:**
```bash
flutter clean                        # Очистить проект
flutter pub get                      # Установить зависимости
flutter run                          # Запустить снова
```

### Ошибка 3: "Android license status unknown"

**Причина:** Не приняты лицензии Android SDK.

**Решение:**
```bash
flutter doctor --android-licenses
```
На все вопросы нажимай `y` (yes).

### Ошибка 4: "Gradle plugin version too low"

**Причина:** Устаревшая версия Android Gradle Plugin.

**Решение:**
1. Открой файл `android/settings.gradle.kts`
2. Убедись, что версия не ниже `8.1.1`:
   ```kotlin
   id("com.android.application") version "8.11.1" apply false
   ```
3. При необходимости обнови версию и выполни:
   ```bash
   flutter clean
   flutter pub get
   ```

### Ошибка 5: "SDK not found" или "ANDROID_HOME not set"

**Причина:** Flutter не может найти Android SDK.

**Решение:**

1. Узнай путь к Android SDK:
   - В Android Studio: **File** → **Settings** → **Appearance & Behavior** → **System Settings** → **Android SDK**
   - Обычно это: `C:\Users\<ИМЯ_ПОЛЬЗОВАТЕЛЯ>\AppData\Local\Android\Sdk`

2. Добавь переменную среды:
   - Нажми `Win + R`, введи `sysdm.cpl`
   - **"Дополнительно"** → **"Переменные среды"**
   - В **"Системные переменные"** нажми **"Создать"**
   - Имя: `ANDROID_HOME`
   - Значение: `C:\Users\<ТВОЁ_ИМЯ>\AppData\Local\Android\Sdk`
   - Нажми **OK**

3. Перезапусти командную строку и проверь:
   ```bash
   flutter doctor
   ```

### Ошибка 6: "Java not found" или "JDK not found"

**Причина:** Не установлена Java Development Kit.

**Решение:**

1. Скачай JDK: https://www.oracle.com/java/technologies/downloads/
2. Рекомендую **JDK 17** или **JDK 21**
3. Установи и добавь в PATH переменную `JAVA_HOME`
4. Проверь:
   ```bash
   java -version
   ```

### Ошибка 7: Эмулятор работает очень медленно

**Причина:** Не включена виртуализация.

**Решение:**

1. Перезагрузи компьютер и войди в **BIOS** (обычно F2, F12, Del при загрузке)
2. Найди настройку **Intel VT-x** или **AMD-V**
3. Включи **Virtualization Technology**
4. Сохрани и перезагрузись

Или используй **Genymotion** — он быстрее стандартного эмулятора.

### Ошибка 8: "Application not configured for Android"

**Причина:** Не инициализирован Android-проект.

**Решение:**
```bash
flutter create .
flutter pub get
flutter run
```
