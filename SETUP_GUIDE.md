## 🎯 О проекте

**Student Connect (СтудХаб)** - мобильная платформа для студентов, объединяющая:
- 📱 Социальную сеть для студентов
- 🚀 Портфолио проектов
- 💬 Мессенджер для общения
- 🔍 Поиск единомышленников

### Архитектура

```
┌─────────────────┐
│  Flutter App    │  ← Android приложение
│  (Mobile)       │
└────────┬────────┘
         │ HTTPS
         ▼
┌─────────────────┐
│  Backend API    │  ← Node.js + Express
│  (Render.com)   │     https://student-connect-backend.onrender.com
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  SQLite DB      │  ← База данных на сервере
└─────────────────┘
```

### Технологии

**Frontend:**
- Flutter 3.41.6
- Dart 3.0+
- Riverpod (state management)
- Dio (HTTP client)
- Material Design 3

**Backend:**
- Node.js + Express
- SQLite (sql.js)
- JWT Authentication
- Multer (file uploads)
- Deployed on Render.com

## 🔧 Установка Flutter

### Windows

#### Шаг 1: Скачивание

1. Перейдите на https://docs.flutter.dev/get-started/install/windows
2. Скачайте Flutter SDK (zip архив)
3. Распакуйте в `C:\src\flutter` (или другую папку без пробелов)

⚠️ **Важно:** НЕ распаковывайте в `C:\Program Files` (требует прав администратора)

#### Шаг 2: Добавление в PATH

1. Нажмите `Win + R`, введите `sysdm.cpl`
2. Вкладка **"Дополнительно"** → **"Переменные среды"**
3. В разделе **"Переменные пользователя"** найдите `Path`
4. Нажмите **"Изменить"** → **"Создать"**
5. Добавьте: `C:\src\flutter\bin`
6. Нажмите **OK** везде

#### Шаг 3: Проверка

Откройте **новую** командную строку:

```bash
flutter doctor
```

Вы увидите список компонентов. Не пугайтесь крестиков - сейчас исправим!

## 📱 Установка Android Studio

### Шаг 1: Установка

1. Скачайте с https://developer.android.com/studio
2. Запустите установщик
3. Выберите компоненты:
   - ✅ Android Studio
   - ✅ Android SDK
   - ✅ Android Virtual Device (AVD)

### Шаг 2: Настройка SDK

1. Откройте Android Studio
2. **Tools** → **SDK Manager**
3. Вкладка **SDK Platforms:**
   - ✅ Android 14.0 (API 34) или новее
4. Вкладка **SDK Tools:**
   - ✅ Android SDK Build-Tools
   - ✅ Android SDK Command-line Tools
   - ✅ Android SDK Platform-Tools
5. Нажмите **Apply** → **OK**

### Шаг 3: Создание эмулятора

1. **Tools** → **Device Manager**
2. Нажмите **"Create Device"**
3. Выберите устройство (например, **Pixel 5**)
4. Выберите системный образ: **API 34 (Android 14)**
5. Нажмите **Finish**

### Шаг 4: Проверка

```bash
flutter doctor --android-licenses    # Принять лицензии
flutter doctor                        # Проверить установку
```

Должно быть:
- ✅ Flutter
- ✅ Android toolchain
- ✅ Android Studio

## 🚀 Запуск приложения

### Вариант 1: Эмулятор Android

#### Шаг 1: Запустите эмулятор

```bash
flutter emulators                     # Список эмуляторов
flutter emulators --launch <id>       # Запустить
```

Или через Android Studio: **Tools** → **Device Manager** → ▶️

#### Шаг 2: Клонируйте проект (если еще не сделали)

```bash
git clone https://github.com/Mustafa-java/student-connect-app.git
cd student-connect-app
```

#### Шаг 3: Установите зависимости

```bash
flutter pub get
```

#### Шаг 4: Запустите приложение

```bash
flutter run
```

Приложение соберется и запустится на эмуляторе (~2-3 минуты первая сборка).

### Вариант 2: Физическое Android устройство

#### Шаг 1: Включите режим разработчика

1. **Настройки** → **О телефоне**
2. Нажмите 7 раз на **"Номер сборки"**
3. Появится сообщение "Вы стали разработчиком"

#### Шаг 2: Включите отладку по USB

1. **Настройки** → **Для разработчиков**
2. Включите **"Отладка по USB"**
3. Включите **"Установка через USB"** (если есть)

#### Шаг 3: Подключите устройство

1. Подключите телефон к компьютеру через USB
2. На телефоне разрешите отладку (всплывающее окно)
3. Проверьте подключение:

```bash
flutter devices
```

Должно показать ваше устройство.

#### Шаг 4: Запустите приложение

```bash
cd student-connect-app
flutter run -d <device-id>
```

Или просто `flutter run` если устройство одно.

### Вариант 3: Отладка по WiFi (без кабеля)

Если устройство и компьютер в одной WiFi сети:

```bash
adb tcpip 5555                        # На устройстве через USB
adb connect <IP-адрес-телефона>:5555  # Подключиться по WiFi
flutter run                           # Запустить
```

## 🌐 Backend сервер

### Облачный сервер (Production)

Backend уже развернут и работает:

- **URL:** https://student-connect-backend.onrender.com
- **Платформа:** Render.com (бесплатный план)
- **Регион:** Frankfurt, EU
- **База данных:** SQLite (хранится на сервере)

#### Проверка работы:

```bash
curl https://student-connect-backend.onrender.com/
```

Должен вернуть JSON:
```json
{
  "name": "Student Connect API",
  "version": "1.0.0",
  "status": "running"
}
```

#### ⚠️ Важно про бесплатный план:

- Сервер **засыпает** после 15 минут неактивности
- Первый запрос после сна занимает **30-50 секунд**
- **Перед демонстрацией:** откройте приложение за 1-2 минуты

### Локальный backend (для разработки)

Если хотите запустить backend локально:

#### Шаг 1: Установите Node.js

Скачайте с https://nodejs.org/ (LTS версия)

#### Шаг 2: Запустите backend

```bash
cd backend
npm install                           # Установить зависимости
npm start                             # Запустить сервер
```

Сервер запустится на `http://localhost:3000`

#### Шаг 3: Измените URL в приложении

Откройте `lib/services/api_service.dart` и измените:

```dart
// Было:
static const String _baseUrl = 'https://student-connect-backend.onrender.com';

// Стало:
static const String _baseUrl = 'http://10.0.2.2:3000';  // Для эмулятора
// или
static const String _baseUrl = 'http://192.168.X.X:3000';  // Для физического устройства
```

⚠️ **Для эмулятора:** используйте `10.0.2.2` (это localhost эмулятора)
⚠️ **Для физического устройства:** используйте IP вашего компьютера в локальной сети

#### Шаг 4: Пересоберите приложение

```bash
flutter run
```

## 🐛 Частые ошибки и решения

### 1. "No devices found"

**Проблема:** Flutter не видит устройства

**Решение:**
```bash
flutter devices                       # Проверить устройства
flutter emulators                     # Список эмуляторов
flutter emulators --launch <id>       # Запустить эмулятор
```

Для физического устройства:
- Проверьте USB кабель
- Включите "Отладка по USB"
- Разрешите отладку на телефоне

### 2. "Build failed" / Gradle ошибки

**Проблема:** Ошибка при сборке

**Решение:**
```bash
flutter clean                         # Очистить кэш
flutter pub get                       # Переустановить зависимости
flutter run                           # Запустить снова
```

### 3. "Connection refused" / "Network error"

**Проблема:** Приложение не может подключиться к backend

**Решение:**

1. Проверьте, что backend работает:
```bash
curl https://student-connect-backend.onrender.com/
```

2. Если сервер спит, подождите 30-50 секунд

3. Проверьте интернет на устройстве

4. Проверьте URL в `lib/services/api_service.dart`

### 4. Google Fonts не загружаются

**Проблема:** Ошибки загрузки шрифтов в логах

**Решение:** Это не критично - приложение будет использовать системные шрифты. Проверьте интернет-соединение.

### 5. "Android license status unknown"

**Проблема:** Не приняты лицензии Android SDK

**Решение:**
```bash
flutter doctor --android-licenses     # Принять все лицензии
```

Нажимайте `y` на все вопросы.

### 6. "Gradle task assembleDebug failed"

**Проблема:** Ошибка Gradle

**Решение:**

1. Проверьте версию Java:
```bash
java -version                         # Должна быть 11 или выше
```

2. Очистите Gradle кэш:
```bash
cd android
./gradlew clean                       # Linux/Mac
gradlew.bat clean                     # Windows
cd ..
flutter run
```

## 🎯 Тестирование приложения

### Первый запуск

1. **Регистрация:**
   - Откройте приложение
   - Нажмите "Регистрация"
   - Заполните форму (имя, email, пароль, университет)
   - Выберите навыки
   - Нажмите "Зарегистрироваться"

2. **Главная лента:**
   - Просмотрите посты других пользователей
   - Лайкните пост
   - Откройте комментарии

3. **Создание поста:**
   - Нажмите "+" внизу
   - Выберите фото
   - Добавьте описание и теги
   - Опубликуйте

4. **Проекты:**
   - Перейдите в "Поиск"
   - Найдите проект
   - Откройте детали
   - Скачайте ZIP (если есть)

5. **Чаты:**
   - Перейдите в "Сообщения"
   - Найдите пользователя
   - Напишите сообщение

6. **Профиль:**
   - Откройте свой профиль
   - Нажмите "Редактировать"
   - Измените аватар и биографию
   - Сохраните
